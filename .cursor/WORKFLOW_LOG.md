# KnitLens Cursor Workflow Log

## Purpose

This log is a **human- and agent-readable history** of important multi-step workflows in this project.  
It is not a strict audit trail, but a lightweight way to answer:

- Which request started this workflow?
- Which subagent(s) and skills were involved?
- What major artifacts (rules, skills, hooks, subagents, code scaffolds) were created or changed?

## Logging Architecture

Workflow logging is automated via a **hook + script + JSON** pipeline:

- Agents write a structured JSON description of the completed workflow to:  
  `.cursor/last-workflow.json`
- Cursor project hooks (configured in `.cursor/hooks.json`) trigger on events such as:
  - `sessionEnd`
  - `subagentStop`
- These hooks execute:  
  `node .cursor/hooks/append-workflow-log.js`
- The script:
  - Reads `.cursor/last-workflow.json`
  - Formats a Markdown entry
  - Appends it to `.cursor/WORKFLOW_LOG.md`
  - Deletes `.cursor/last-workflow.json`

Important:

- Agents **MUST NOT** edit `WORKFLOW_LOG.md` directly.
- All new entries **SHOULD** be generated by writing `.cursor/last-workflow.json` and letting the hook pipeline append them.

## Usage Guidelines

- Use this log **only for meaningful, multi-step workflows**, not for every tiny edit.
- The **Main Agent** is responsible for deciding when to create `.cursor/last-workflow.json` (see `workflow-logging.md`).
- Subagents **may** contribute structured metadata to the Main Agent but should not modify this Markdown file.

## Entry Template

Use this structure for each entry:

```text
## YYYY-MM-DD – <Short workflow title>

- **User intent**: <Concise description of the original request or problem.>
- **Main Agent summary**: <1–2 sentences describing the chosen approach.>
- **Subagents involved**:
  - Screen Implementer: <yes/no + short note if yes>
  - AI Route Implementer: <yes/no + short note if yes>
  - Retro UI Reviewer: <yes/no + short note if yes>
  - Supabase Data Agent: <yes/no + short note if yes>
- **Skills used**: <List of SKILLs used, if relevant.>
- **Key artifacts**:
  - Rules: <created/updated rule files, if any>
  - Skills: <created/updated skills, if any>
  - Hooks: <created/updated hooks, if any>
  - Subagents: <created/updated subagent definitions, if any>
  - Code scaffolds: <high-level mention only, no large code dumps>
- **Open questions / follow-ups**: <What still needs a decision or implementation?>
```

## Notes and Limitations

- This log is updated automatically by the hook + script pipeline based on `.cursor/last-workflow.json`.
- If the logging architecture changes in the future, this file can be:
  - Replaced by a more formal mechanism, or
  - Used as a human-readable index that points to richer logs.
