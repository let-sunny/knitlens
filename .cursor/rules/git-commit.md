# Git Commit Rules

All commits generated by agents must follow this format.

type(scope): summary

## Allowed Types

feat
fix
refactor
docs
chore
test

## Examples

feat(project): add project creation screen
feat(ai): implement pattern analysis route
fix(tracker): correct step toggle state
docs(architecture): update data model
chore: update dependencies

## Guidelines

Use present tense.

Keep summaries short and descriptive.

Do not use vague messages such as:

update
fix stuff
changes

Commit messages should clearly describe the change.
