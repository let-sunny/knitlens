## Added

### `/projects/[id]/clarify` — 명확화 질문 페이지

- 프로젝트/패턴에 대해 AI가 생성한 clarification questions를 로드해 표시.
- 질문 타입에 따라 입력 방식 지원:
  - `choice`: 라디오 버튼 옵션.
  - `number`: 숫자 입력.
  - `text`: 일반 텍스트 입력.
- 사용자가 모든 질문에 답한 뒤 **Submit**을 누르면 명확화 API를 호출:
  - 추가 질문이 있으면 동일 화면에서 새 질문 세트를 표시.
  - 더 이상 질문이 없으면 패턴 컴파일 API를 호출 후 `/projects/[id]`로 이동.
- 에러 상태:
  - 필수 답변 누락, 숫자 입력 오류.
  - 명확화/컴파일 API 실패 시 사용자 친화 메시지 표시.

### `POST /api/pattern/clarify` — 명확화 루프 API

- 요청: `patternId`, `answers: Answer[]` (`questionId`, `value`).
- 동작:
  - Project/Pattern 및 원본 패턴 텍스트 조회.
  - LLM에 패턴 텍스트 + 사용자의 clarification answers를 전달해 추가 질문 필요 여부 판단.
  - LLM 응답을 JSON-only + 스키마(`questions: Question[]`)로 검증.
  - 유효한 답변을 `clarification_answers` 테이블에 저장.
- 응답:
  - `{ "questions": Question[] }` — 추가 질문이 없으면 빈 배열 반환.
- 에러:
  - 잘못된 요청: `error: "invalid_clarification_payload"`.
  - LLM 호출/검증/저장 실패: `error: "clarification_failed"`.

### `POST /api/pattern/compile` — 최종 패턴 컴파일 API

- 요청: `patternId`, `answers: Answer[]`.
- 동작:
  - Project/Pattern 및 원본 패턴 텍스트 조회.
  - LLM에 패턴 텍스트 + 모든 clarification answers를 전달해 최종 Section/Row/Step 구조 생성.
  - LLM 응답을 JSON-only + 스키마로 검증:
    - `patternId: string`.
    - `sections: Section[]` (Section/Row/Step 구조).
  - `patterns.sections`를 갱신하고 `projects.status`를 `compiled`로 업데이트.
- 응답:
  - `{ "patternId": string, "sections": Section[] }`.
- 에러:
  - 잘못된 요청: `error: "invalid_compile_payload"`.
  - 패턴 텍스트 누락, LLM 호출/검증 실패: `error: "pattern_compilation_failed"`.

### Supabase

- **스키마**: `clarification_answers` 테이블 추가 (마이그레이션: `supabase/migrations/00002_add_clarification_answers.sql`).
  - `project_id`, `pattern_id`, `question_id`, `value`, `created_at`.
- **서버 헬퍼**:
  - `lib/supabase/clarification-answers.ts` — clarification_answers insert 헬퍼.
  - `lib/supabase/patterns.ts` — `getPatternById`, `updatePatternSections` 추가.

### AI 통합

- `lib/llm/prompt-clarify.ts`
  - Clarification 전용 system 프롬프트 및 user 프롬프트 빌더.
  - 출력: `{ "questions": Question[] }`.
- `lib/llm/prompt-compile.ts`
  - Pattern compilation 전용 system 프롬프트 및 user 프롬프트 빌더.
  - 출력: `{ "patternId": string, "sections": Section[] }`.
- `lib/llm/schemas.ts`
  - Row/Step 구조를 포함하는 Section/Row/Step 런타임 스키마 강화.
  - `validateClarifyOutput`, `validateCompileOutput` 추가.

## Known limitations / Follow-ups

- `/projects/[id]` 트래커 UI, `GET /api/projects/[id]`, `POST /api/projects/[id]/progress`는 여전히 후속 스펙.
- Clarification answers는 저장되지만, 재분석/리포트 기능은 아직 제공되지 않음.
- LLM 프롬프트/스키마는 MVP 수준으로 설계되어 있으며, 실제 패턴 샘플에 맞춰 추가 튜닝이 필요할 수 있음.
