# Changelog: v0.3.0 — Project Tracker

**날짜**: 2026-03-12  
**스펙 문서**: `.cursor/plans/project_tracker_spec_prd_dfcfd5ac.plan.md` (PRD + 구현 플랜)

## Added

### `/projects/[id]` — 프로젝트 트래커 화면

- 컴파일된 패턴의 섹션·스텝 목록 표시.
- 스텝 완료 토글, 반복 횟수(repeatCount) 입력·갱신.
- 진행 상태 유지(재접속 시 복원).

### `GET /api/projects/[id]` — 프로젝트·패턴·진행 상태 조회

- path param `id`로 프로젝트 조회. 없으면 404.
- `project.status`가 `compiled`가 아니면 404.
- project, pattern(sections 포함), progress 반환. progress 없으면 초기 생성 후 반환.

### `POST /api/projects/[id]/progress` — 스텝 완료·반복 횟수 갱신

- 요청 본문: `stepId`, `completed`, `repeatCount`.
- 해당 project의 progress 갱신 후 갱신된 progress 반환.
- 에러 시 `progress_update_failed` 등 사용자 친화 메시지.

### `project_progress` 테이블 및 서버 CRUD

- 마이그레이션: `project_progress` (id, project_id, current_section_id, current_row_id, current_step_id, steps JSONB, created_at, updated_at).
- 서버 전용: progress 생성(최초 진입 시)/조회/갱신.
