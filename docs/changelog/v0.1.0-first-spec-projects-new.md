## Added

### `/projects/new` — 새 프로젝트 생성 페이지

- 프로젝트 제목 입력, 패턴 PDF 업로드(파일 선택), 선택적 사이즈 입력.
- **Analyze Pattern** 실행 시:
  - 프로젝트 생성 → PDF Storage 업로드 → 서버에서 PDF 텍스트 추출 → `POST /api/pattern/analyze` 호출.
  - 분석 결과에 따라 자동 이동:
    - 질문 없음 → `/projects/[id]`
    - 질문 있음 → `/projects/[id]/clarify`
- 에러 시 사용자 메시지: PDF 미첨부, 잘못된 파일 형식, 텍스트 추출 실패, 분석 실패 등.

### `POST /api/pattern/analyze` — 패턴 텍스트 분석 API

- 요청: `projectId`, `patternText`.
- 응답: `patternId`, `sections`, `questions`.
- JSON 전용, 런타임 스키마 검증, BYOK LLM(OpenAI).
- 성공 시 Project 상태 갱신(`clarification` 또는 `compiled`), Pattern·clarification_questions 저장.

### Supabase

- **스키마**: `projects`, `pattern_sources`, `patterns`, `clarification_questions` 테이블 (마이그레이션: `supabase/migrations/00001_initial_schema.sql`).
- **Storage**: `pattern-pdfs` 버킷 사용 (설정 방법: `supabase/README.md`).
- **서버 헬퍼**: `lib/supabase/server.ts`, `projects.ts`, `pattern-sources.ts`, `patterns.ts`, `storage.ts`.

### PDF 텍스트 추출

- 서버 전용: `lib/pdf/extract-text.ts` (pdf-parse 사용).
- 업로드: `lib/supabase/storage.ts` — FormData → Storage 경로 반환.

### UI

- **system.css**: `@sakun/system.css` 적용 (레트로 맥 스타일).
- `/projects/new`: window/title-bar/btn/field-row 등 system.css 클래스 사용.
- `/projects/[id]`, `/projects/[id]/clarify`: 플레이스홀더 페이지 (후속 스펙에서 본문 구현).

## Known limitations / Follow-ups

- `/projects/[id]`, `/projects/[id]/clarify`: 플레이스홀더만 있음. 트래커·명확화 UI는 후속 스펙.
- `POST /api/pattern/clarify`, `POST /api/pattern/compile` 미구현.
- 프로젝트 목록/홈에서 `/projects/new`로 가는 진입점은 필요 시 별도 작업.
- Supabase·OpenAI 환경 변수 설정 필요 (`.env.local.example` 참고).
