# Pull Request Guidelines

This document describes how pull requests should be structured in the KnitLens repository.

The goal is to make changes understandable and traceable.

## Pull Request Structure

Every PR should include the following sections.

### Summary

Brief explanation of what this PR introduces.

### Context

Why this change is needed.

### Changes

List the main changes included in the PR.

Example

- implement project creation screen
- add pattern analysis API route
- add Supabase project model

### Related Docs

Reference relevant documentation if applicable.

Examples

docs/product/one-pager.md  
docs/architecture/screen-flow.md  
docs/architecture/data-model.md  

### Follow-ups

Future improvements or work that will come later.
