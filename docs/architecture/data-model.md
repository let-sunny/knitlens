# Data Model

## Core Entities

### Project

Represents a knitting project created by the user.

Project

- id: string
- title: string
- patternPdfUrl: string
- rawPatternText: string
- status: "draft" | "analyzing" | "clarification" | "compiled"
- createdAt: string (ISO datetime)
- updatedAt: string (ISO datetime)

Relationships

Project
└─ Pattern
└─ Sections[]
└─ Rows[]
└─ Steps[]

### Pattern

Represents the compiled knitting pattern generated by the AI.

Pattern

- id: string
- projectId: string
- title: string
- notes: string | null
- sections: Section[]

Example

```json
{
  "id": "pattern_1",
  "projectId": "project_1",
  "title": "Basic Raglan Sweater",
  "notes": "Converted from original PDF pattern",
  "sections": []
}
```

### Section

Logical grouping of rows.

Examples

- Body
- Sleeves
- Collar

Section

- id: string
- title: string
- order: number
- rows: Row[]

Example

```json
{
  "id": "section_body",
  "title": "Body",
  "order": 1,
  "rows": []
}
```

### Row

Represents a single row or round in the knitting pattern.

Row

- id: string
- order: number
- instructions: string
- repeat: number | null
- notes: string | null
- steps: Step[]

Example

```json
{
  "id": "row_1",
  "order": 1,
  "instructions": "K2, P2 across",
  "repeat": null,
  "notes": null,
  "steps": []
}
```

### Step

Smallest unit of instruction.

Step

- id: string
- order: number
- instruction: string
- stitchCount: number | null

Example

```json
{
  "id": "step_1",
  "order": 1,
  "instruction": "Knit 2 stitches",
  "stitchCount": 2
}
```

### PatternSource

Represents the uploaded pattern source.

PatternSource

- id: string
- projectId: string
- fileType: "pdf"
- fileUrl: string
- extractedText: string

### ProjectProgress

Represents the user's progress for a compiled project on the tracker screen.

ProjectProgress

- id: string
- projectId: string
- currentSectionId: string | null
- currentRowId: string | null
- currentStepId: string | null
- steps: StepProgress[]

StepProgress

- stepId: string
- completed: boolean
- repeatCount: number | null

Example

```json
{
  "id": "progress_1",
  "projectId": "project_1",
  "currentSectionId": "section_body",
  "currentRowId": "row_1",
  "currentStepId": "step_1",
  "steps": [
    {
      "stepId": "step_1",
      "completed": false,
      "repeatCount": null
    }
  ]
}
```

### ClarificationQuestion

Represents a single clarification question generated by the AI.

ClarificationQuestion

- id: string
- patternId: string
- text: string
- type: "choice" | "number" | "text"
- options: string[] | null

### ClarificationAnswer

Represents a user answer to a clarification question.

ClarificationAnswer

- id: string
- projectId: string
- patternId: string
- questionId: string
- value: string | number

## Project Status State Machine

Projects move through the following lifecycle.

draft  
↓ upload PDF  
analyzing  
↓ pattern analyzed  
clarification (optional)  
↓ clarification answered  
compiled

State descriptions

draft  
Project created but analysis has not started.

analyzing  
AI is extracting structure from the pattern.

clarification  
AI requires additional answers from the user.

compiled  
Pattern structure is finalized and ready for tracking.
