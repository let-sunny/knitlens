# Screen: /projects/[id]/clarify

## Purpose

Resolve ambiguity found during pattern analysis before final pattern compilation.

This screen maps to the clarification step in the screen flow:

Create Project
→ Pattern Analysis
→ Clarification (optional)
→ Pattern Compilation
→ Project Tracker

## Inputs

- clarification questions generated by AI
- available answer options
- optional freeform answer if supported later

## Sections

- page title
- clarification summary
- question list
- answer controls
- continue button

## Primary Actions

- Submit Answers
- Continue

## Action Behavior

### Submit Answers / Continue

1. submit user answers to clarification processing
2. determine whether more clarification is needed
3. if clarification is complete, continue to pattern compilation
4. redirect to project tracker when compilation is complete

## API Dependencies

- `POST /api/pattern/clarify`
- `POST /api/pattern/compile`

## Success Paths

### If more clarification is needed

stay on:

- `/projects/[id]/clarify`

with updated questions

### If clarification is complete

redirect to:

- `/projects/[id]`

## Error States

- missing answer
- invalid clarification payload
- clarification response validation failure
- pattern compilation failure

## Notes

- This screen should focus on decision-making, not full project tracking.
- The user should clearly understand what is ambiguous and what decision is being requested.
- The screen should not directly edit raw pattern text.
